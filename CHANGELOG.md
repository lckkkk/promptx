# Changelog

## 0.1.40

- 任务列表新增“未读”标识：当任务运行进入终态后，会在左侧列表上显示未读点；打开任务后会自动标记为已读，并把状态按用户持久化到服务端，避免刷新后丢失。
- 修复未读状态的两处边界问题：首次产生终态 run、但还没有任何已读记录的任务，现在也会正确标记为未读；同时修复打开任务后详情状态轻微滞后时没有真正落库已读、导致任务反复显示未读的问题。
- 代码变更能力补齐多仓库场景：当项目绑定的是父目录、其下包含多个 Git 子仓库时，“当前变更 / 任务累计 / 本轮”现在都能继续读取和展示子仓库 diff，不再只支持单一根仓库。
- 修复多仓库代码变更中的前端回归：解决 `selectedFilePath is not defined` 报错，以及“任务累计 / 本轮”面板错误提示“当前目录不是 Git 仓库”导致无法查看 diff 的问题。
- 新用户首次登录体验收口：首次进入工作台时不再默认自动创建一个空任务，只保留用户主动新建任务的入口。
- 通知消息补全项目上下文：任务通知内容里增加项目名称，方便在多个项目同时运行时区分来源；本地更新成功后的重启通知也统一发送到“lc 提醒”通知配置。
- 继续打磨任务列表样式与交互：收紧项目筛选框尺寸和边框样式，调整项目名称 tag 的视觉表现，并清理任务卡里项目 tag 左侧多余色块、圆点和不协调的胶囊感。
- 任务输入区交互优化：编辑器中按 `Enter` 可直接发送当前任务输入，减少桌面端反复切按钮的成本。
- 本地更新链路补充落地验证：`local:update` 继续沿用“build -> install -> 空闲后重启”，并补强重启状态查询、日志记录与成功通知链路，便于确认当前页面是否已切到最新版本。

## 0.1.39

- 任务与项目工作流继续收口：新建任务改为必须先选项目，不再默认绑定；首页补齐“项目”入口，任务列表顶部布局、按钮占位和手机端展示同步优化。
- 任务列表增强项目维度操作：项目筛选支持多选，项目名称颜色区分进一步拉开，同时收敛任务卡里的项目标签样式，去掉多余色块、圆胶囊和拖拽按钮之间的圆点。
- 待办流转体验增强：待办支持多选批量添加回输入区；当编辑区已有内容时，改为先弹窗确认“追加”而不是直接覆盖；批量添加时会自动补序号，并按“分号 + 换行”组织文本，不再插入多余空行。
- 手机端任务列表补齐右滑删除能力，并处理与浏览器返回手势的冲突，避免在列表里误触返回；同时修复移动端任务卡内容被挤压、右侧大片空白等布局问题。
- 登录与用户体系升级：禁用默认 `default` 直入模式，改为必须使用账号密码登录；同时修复 npm 安装包遗漏用户管理脚本的问题，保证 `promptx user add/list/remove/reset-password` 等命令在全局安装场景下可用。
- 通知能力重构为“设置统一管理”：设置页新增“通知配置”目录，支持通知配置的增删改查与“设为默认”；任务编辑弹窗只保留通知配置选择；新建任务会自动继承默认通知配置；删除默认通知配置时会自动清空系统默认值。
- 项目管理继续增强：新建项目选择工作目录时支持新建目录；任务通知配置与项目选择、执行引擎等能力的心智进一步统一为“任务 -> 项目 -> 目录 -> 执行引擎线程”。
- 本地更新链路重构：新增 `npm run local:update`，统一执行 build、全局安装与“空闲后自动重启”；重启完成后会写状态文件、支持页面 toast，并可通过指定通知配置发送成功通知；同时补充 `CLAUDE.md` 规范，要求在影响本地可见行为的改动后按需执行 `local:update`。
- 修复 planned restart / 空闲重启的多处细节问题：避免因先 stop 导致 runner 失联提示误报，支持后台守护等待活动 run 清空后再重启，并增强重启结果反馈与通知投递日志。
- 输入区新增 `/` 命令提示：在编辑器中输入 `/` 时，会按当前项目绑定的执行引擎显示对应命令建议，支持键盘上下选择与 Enter 插入，方便快速调用 Codex / Claude Code / OpenCode 的常用命令。

## 0.1.38

- 修复 npm 安装包缺少 `scripts/user-manager.mjs` 的发布配置问题，避免用户安装后执行 `promptx user add <username>` 等用户管理命令时直接报 `MODULE_NOT_FOUND`。
- 补齐用户管理 CLI 脚本随包发布，确保 `promptx user add/list/remove/reset-password` 在全局安装场景下可正常使用。

## 0.1.34

- 统一图片预览层遮罩，直接复用和工作台弹窗一致的 `theme-modal-backdrop`，不再单独维护一套 overlay 背景，避免预览层与其它弹窗视觉割裂。
- 修复放大图片时顶部工具按钮对比度不足的问题：为缩放、翻页和关闭按钮补上稳定的底色、边框和阴影，在亮图背景上也能清楚辨认。

## 0.1.33

- 修复安装包在 Node.js 22 下启动失败的问题：移除 `jimp` 及其 `@jimp/plugin-print -> simple-xml-to-json` 这条有问题的 ESM 依赖链，避免 `promptx start` 在加载阶段直接抛出 `does not provide an export named 'default'`。
- 上传图片改为使用现有的 `@napi-rs/canvas` 完成读取、缩放和 JPEG 导出，不再依赖 `jimp`，同时保持上传后图片会统一缩放到 `1600x1600` 以内的现有语义。
- 补充图片上传回归测试，并更新锁文件与打包产物校验，确保新安装包不再带入这条启动时会炸掉的依赖链。

## 0.1.32

- 继续收敛执行过程面板：统一过程事件卡片为白底轻边框，过程区与输入区统一等宽字体，移动端优先折行显示，减少横向滚动与样式不一致带来的阅读负担。
- 精简子代理展示：`Codex / Claude Code / OpenCode` 的子代理事件统一为更轻量的一行摘要，移动端最多显示两行、桌面端单行；同时去掉多余的“更新”标签、局部展开/收起和重色块背景。
- 下线执行过程中的代码类自动识别兼容：移除 `code_text / code_snippet / numbered_lines / build_error / search_results` 等旧分支，统一回退为稳定的文本或少量明确结构块，减少误判和错误代码块渲染。
- 优化前端打包输出：为 `tiptap`、图标与 markdown 等依赖拆分 vendor chunk，消除 Vite 的大包告警，降低工作台主包体积。

## 0.1.31

- 继续打磨执行过程展示：统一 `Codex / Claude Code / OpenCode` 的终端输出、搜索命中、构建报错与 diff 渲染，补齐移动端横向滚动、代码字体统一和多余元信息收敛，整体阅读更稳定。
- 修复执行过程中的多处展示问题：去掉“目标”“退出码 0”“完成 reasoning”等低价值噪音，长命令与路径改为更紧凑的 mono 展示，单条搜索结果、编号源码、build error 与混合 prose/code 内容都能更一致地渲染。
- 加固执行过程展开状态与文本解析：避免 block 重排后展开/收起状态串位；过程文本不再错误吞掉前导缩进；同时补齐相关单测与构建验证。

## 0.1.30

- 工作台编辑器升级为基于 Tiptap 的图文混排块编辑器，支持图片、文本文件与 PDF 导入；修复输入、焦点、滚动与移动端布局问题，并清理旧编辑器实现。

## 0.1.29

- 重做执行过程 detail 展示：新增结构化解析与渲染，针对 `Codex / Claude Code / OpenCode` 的过程事件统一支持待办列表、代码块、行号片段、XML-like 输出、编号列表、Markdown 表格/列表等内容，排版更稳定、更接近原生 CLI。
- 收敛执行过程样式细节：统一过程卡片、胶囊与详情区的字体、代码字体、长命令换行、圆角与主题表现；隐藏“已连接项目 / 会话已创建 / 已返回结果 / 本轮执行结束”等低价值系统噪音，提升过程区可读性。
- 修复 Claude / OpenCode 的待办展示：不再只显示“待办列表连续更新 N 次”，改为展示真实 checklist；同时补齐 Claude 401 认证错误识别与 fail-fast，避免在 PromptX 中一直卡住无反馈。
- 修复 Codex 同一轮运行中待办卡片重复且状态滞后的问题：同一轮执行现在只保留最新一张待办快照，避免用户看到旧的 `2/3` 卡片，最终完成态能稳定显示为 `3/3`。

## 0.1.28

- 项目管理新增“新会话”能力：可一键清空当前项目绑定的线程/会话信息，并在下一次运行时从全新会话重新开始，行为更接近 Codex / Claude Code / OpenCode 里的 `/new`。
- “新会话”会同时清空该项目下所有任务的执行历史，但保留项目本身、工作目录、执行引擎配置以及右侧编辑区内容，避免新旧会话结果混在一起。
- 运行中的项目会被禁止执行“新会话”，并补充前后端提示、确认弹窗、中英双语文案与自动化测试，确保这条重置链路可用且稳定。

## 0.1.27

- 左侧任务列表新增桌面端拖拽排序能力，并改用 `vue-draggable-plus` 接管交互，拖拽手感、占位反馈与排序稳定性更好。
- 任务排序改为服务端持久化：拖拽后的顺序会写入数据库，刷新页面、重启服务或远端页面同步后都能保持一致。
- 修复任务排序当前页不立即生效的问题：任务列表合并逻辑改为以服务端返回顺序为准，不再被本地旧顺序覆盖。
- 补充任务重排接口校验与相关前后端测试，同时收敛本地重排后的 SSE 冗余刷新，减少一次无意义列表拉取。

## 0.1.26

- 为当前运行任务补充低频状态对账：即使 SSE 漏掉终态事件，前端也会定时重新拉取当前任务的运行历史和项目状态，避免页面长时间停留在“运行中”，只有刷新后才恢复真实结果。

## 0.1.25

- 中栏新增执行过程显示开关，默认显示；用户可通过眼睛按钮按本地偏好隐藏“执行过程”卡片，只保留 PromptX 输入与模型回复，降低日常查看噪音。
- 修复隐藏执行过程后的“有新消息，跳到底部”判断：当过程区已隐藏时，后台仅有执行事件刷新不再被误判为新消息，也不会无意义打断当前滚动。
- 调整中栏消息头文案，统一收敛为 `PromptX：` 与 `{agent}：`，让执行区更接近对话阅读体验。

## 0.1.24

- 项目管理支持手动填写并复用已有会话 ID，适配从 Codex / Claude Code / OpenCode 外部创建会话后继续在 PromptX 中接管；项目真正运行前仍允许调整，运行后自动锁定，并支持一键复制当前会话 ID。
- 服务端补齐手动绑定会话的状态语义：手填会话 ID 的项目在首次真实运行前不再误判为“已启动”，runner 回写真实线程信息时会自动解除手动绑定标记，避免后续编辑规则混乱。
- 修复“编辑任务”弹窗布局回归，重新理顺左右导航与分区内容结构，保证自动化、通知等设置区块排版稳定。
- 修复“代码变更”弹窗里的“刷新”交互：手动点击时会清掉当前任务的 diff 缓存并强制重新拉取，不再出现看起来像按钮但实际几乎没反应的问题。
- 统一顶部“代码变更”入口按钮样式，与“管理项目”保持一致，减少工具区视觉噪音。

## 0.1.23

- 统一工作台弹窗壳子，并收敛移动端的弹窗、页头、项目选择与目录选择交互；修复手机端项目列表滚动异常，整体视觉与行为更一致。
- 增强编辑区文件导入能力，统一选择文件、拖入文件与粘贴文件的处理路径，补齐更多文本类文件支持，并优化导入后与当前输入焦点、滚动位置的协同体验。
- 修复历史执行过程显示回归：旧 turn 在事件未加载时不再误显示为“本轮没有记录执行过程”，最新与历史 turn 的展开/懒加载判断重新理顺。
- 顶部 Toast 新增 `success / warning / error / info` 四类样式，并按语义调整现有提示；未绑定项目时发送仅提示“请先选择项目”，不再强行弹出项目管理，移动端会同步切到执行面板。
- 修复两处前端组件告警：补齐编辑区文件块使用的 `FileText` 图标引入，并为目录选择弹窗补上 `suggestions` prop 声明。

## 0.1.22

- 修复任务结束后工作台仍反复请求 `tasks`、`workspace-diff` 与当前任务详情接口的问题；收窄 realtime 同步范围，并避免重复终态事件触发无意义刷新。
- 修复发送消息后当前任务被持续判定为“未保存”从而每 1.5 秒重复自动保存的问题；对任务快照中的待办块内容补齐归一化，避免前端临时字段导致脏状态死循环。
- 为任务更新增加服务端幂等保护：当保存内容与当前存储完全一致时，不再更新 `updatedAt`、也不再广播 `tasks.changed`，降低多页面/弱网场景下的连锁刷新风险。

## 0.1.21

- 优化右侧编辑区的焦点与滚动体验：插入图片、导入 Markdown/PDF 等内容后，当前输入框会尽量保持在视野内；可视区域内在不同文本块之间切换焦点时不再意外滚动。
- 调整文件处理与发送反馈：按钮 loading 统一改为图标动效，不再切换文案或撑开布局；并补充编辑区滚动相关的单测与浏览器级 E2E。
- 重做中栏运行态提示：移除会造成抖动的底部状态栏，改为右下角悬浮 `Stop` 按钮，并在按钮内加入动态状态点提示运行中。
- 优化手机端详情页页签区域：去掉多余外层包裹，收紧上下留白并拉满左右空间；同时让移动端的悬浮 `Stop` 更贴近右下角。

- 调整当前任务的后台同步策略：仅在真正切换任务或显式要求时才滚到底部，普通的任务状态刷新不再打断用户正在查看的历史内容。

- 服务端错误返回补齐稳定的 `messageKey`，前端优先按 key 翻译；同时把 Relay 诊断、自动化通知、任务设置等动态文本一并接入 locale，减少英文界面穿透中文的问题。

- 补齐多语言相关自动化验证：修复前端错误格式化回归，更新页面级 E2E 基座，并验证最新 turn 展开、历史执行过程加载和滚动跟随等关键流程未受影响。

## 0.1.13

- 加固 Relay 登录链路：登录与管理登录改为 `POST`，并新增登录限流，降低暴力尝试与口令暴露风险。
- 优化 Relay 相关页面的手机端体验，登录页、管理登录页和统计页在小屏设备上都能更稳定使用。

- 编辑区发送按钮补齐三段状态反馈：`发送`、`发送中`、`执行中`，降低“点了没反应”的疑惑。

## 0.1.12

## 0.1.11

- 加固 Relay 自动重连恢复链路：固定间隔重连升级为指数退避，并在连接成功后重置重连计数，减少 Relay 重启或网络抖动时的无效重试。
- 补齐 Relay client 健康检查与自愈能力：新增心跳超时巡检、假死连接主动重连，以及类似 Mac 休眠/唤醒后的时间跳变检测，避免远程页面断开后必须手动保存设置才能恢复。
- 修复 Relay client 旧连接 `close` 回调误伤新连接的竞态问题，避免在 Windows 等时序下反复出现“旧设备连接被新连接替换”的重连循环。
- 对 `invalid_token`、`invalid_device`、`invalid_tenant` 等不可恢复错误改为暂停自动重连，并在状态里明确暴露暂停原因，方便快速定位配置问题。
- 设置页“远程”增加“立即重连”按钮，并补充等待重连 / 已暂停重连等状态反馈，便于手动触发恢复和远程排障。
- 新增 Relay 重连相关服务端与接口测试，覆盖拒绝后暂停、重连成功清零、心跳超时自愈、休眠恢复近似场景和手动重连。
- 调整左侧任务列表为固定创建顺序，不再因运行、保存、自动刷新或 `updatedAt` 变化跳动；同时修复慢网切换任务时旧草稿标题短暂显示到新任务上的问题。

## 0.1.10

- 重构执行链路为 `server + runner` 双进程架构：`server` 专注 HTTP API、任务/项目存储与 SSE，`runner` 专注 agent 子进程生命周期、事件流、stop/kill/超时回收，显著降低长任务拖垮主服务的风险。
- 新增 runner 全局并发控制与排队能力，默认真实 agent 最大并发提升到 `3`，并支持在“设置 -> 系统”里热更新并发上限；排队中的任务在扩容后可立即启动。
- 补齐运行态诊断与控制面韧性：增加 server/runner 内部 HTTP 超时、queued/stopping 心跳、失联恢复与 stop 原因分类，并在系统设置页展示 active/tracked/queued、终态计数、stop timeout 阶段等实时诊断信息。
- 收口服务端热路径与路由结构，拆分 `tasks / codex / system / internal / assets / webApp` 路由模块，降低 `apps/server/src/index.js` 复杂度，同时把 git diff 等低频重操作继续隔离出热接口。
- 新增真实 E2E 回归覆盖：验证多任务并发、真实长任务 stop/回收，以及浏览器设置页里动态调大并发后 queued 任务立即启动，Codex / OpenCode 两条链路均已跑通。
- 修复编辑区长提示词刷新后高度不自动撑开、以及双滚动条的问题，保证长输入场景下的编辑体验稳定。
- 新增定式任务能力，补齐相关工作流与交互细节，便于按固定结构复用任务模板。
- 收口主题系统与毛玻璃皮肤细节，修复删除确认弹窗按钮不可见、主题项冗余色条、主题名称与说明乱码等问题。
- 新增 `Ink Mist` 主题，补齐独立的冷雾水墨风格，并在移动端默认禁用高开销毛玻璃主题以保证性能稳定。
- 优化项目管理体验，修复“从执行中的项目进入后所有项目都不能删除”的错误禁用逻辑，并让任务列表在刷新、执行和状态变化时保持稳定顺序。
- 修复 Codex 回复区列表样式，恢复有序/无序列表 marker，并补齐嵌套列表层级样式与间距。
- 增强 Relay 稳定性与诊断能力：增加 ping/pong 心跳、超时重连、client/server 关键日志与 recent events，以及本地设置页一键复制诊断信息。
- 新增任务级“代办”收纳箱：支持把编辑区内容暂存为代办、在弹层中统一管理、二次确认后回填或删除，并随任务一起持久化保存。
