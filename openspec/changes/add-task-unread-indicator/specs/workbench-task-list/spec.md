## ADDED Requirements

### Requirement: 任务列表展示执行完成未读状态
任务列表 SHALL 仅在任务运行进入终态且用户尚未查看结果时展示未读标识。

#### Scenario: 其他任务执行完成后显示未读
- **WHEN** 某个非当前查看中的任务运行进入终态
- **THEN** 该任务在列表中显示未读标识
- **AND** 未读标识持续存在，直到用户打开该任务

#### Scenario: 当前正在查看的任务执行完成
- **WHEN** 用户当前已经打开某个任务，且该任务运行进入终态
- **THEN** 该任务不应额外显示未读标识

### Requirement: 未读状态按用户持久化
系统 SHALL 按用户与任务维度持久化未读状态，并保证不同用户互不影响。

#### Scenario: 不同用户查看同一任务
- **WHEN** 用户 A 已查看某任务而用户 B 未查看
- **THEN** 用户 A 不显示该任务未读
- **AND** 用户 B 仍显示该任务未读

#### Scenario: 刷新后保留未读状态
- **WHEN** 任务已产生未读且用户刷新页面或重新登录
- **THEN** 系统仍显示该任务未读

### Requirement: 打开任务后自动标记已读
系统 SHALL 在用户打开任务详情时自动清除该任务的执行完成未读状态。

#### Scenario: 打开未读任务
- **WHEN** 用户打开一个带有未读标识的任务
- **THEN** 系统将该任务标记为已读
- **AND** 任务列表中的未读标识消失

#### Scenario: 非终态运行不触发未读
- **WHEN** 任务仍处于 `queued`、`starting`、`running` 或 `stopping`
- **THEN** 系统不为该任务设置未读标识
