## ADDED Requirements

### Requirement: 工作区代码变更应聚合工作目录下多个 Git 仓库

系统 SHALL 在 `workspace` 级代码变更查询中，聚合当前项目工作目录本身以及其子目录内独立 Git 仓库的变更结果，而不是只返回单一仓库的 diff。

#### Scenario: 工作目录下包含多个子 Git 仓库

- **GIVEN** 一个 PromptX 项目绑定的工作目录本身不是唯一仓库，且其下存在多个独立 Git 仓库
- **WHEN** 用户查看该任务的 `workspace` 代码变更摘要或详情
- **THEN** 系统返回的变更结果应包含所有纳入范围仓库的改动总和
- **AND** 不得只显示其中某一个仓库的变更

### Requirement: 聚合后的文件变更必须带仓库归属信息

系统 SHALL 为多仓库聚合结果中的每个变更文件附带稳定的仓库标识信息，供前端展示和区分。

#### Scenario: 不同子仓库存在同名文件

- **GIVEN** 工作目录下两个独立 Git 仓库都存在同名文件变更，例如 `src/index.js`
- **WHEN** 用户查看代码变更文件列表
- **THEN** 系统返回的文件项必须带有仓库标识
- **AND** 前端必须能区分这些同名文件分别属于哪个仓库

### Requirement: 工作区代码变更摘要应反映聚合后的总数

系统 SHALL 在任务列表和代码变更摘要中展示多仓库聚合后的总文件数、增删行统计。

#### Scenario: 首页展示工作区变更规模

- **GIVEN** 当前任务绑定的工作目录下有多个 Git 仓库发生改动
- **WHEN** 首页或任务列表请求工作区变更摘要
- **THEN** 返回的 `fileCount`、`additions`、`deletions` 应为所有纳入仓库的汇总值
