# Capability: 项目筛选与任务标题优化

## ADDED Requirements

### Requirement: 任务列表支持项目筛选

任务列表顶部应显示项目筛选下拉框，用户可选择查看特定项目的任务。

#### Scenario: 筛选器选项

**Given** 用户有 3 个项目："前端项目"、"后端项目"、"移动端项目"
**When** 打开任务列表页面
**Then** 筛选器下拉框显示："全部项目"、"前端项目"、"后端项目"、"移动端项目"

#### Scenario: 筛选后任务列表更新

**Given** 用户有 10 个任务，其中 4 个绑定到"前端项目"
**When** 在筛选器中选择"前端项目"
**Then** 任务列表只显示这 4 个任务

#### Scenario: 全部项目选项

**Given** 筛选器当前选中"前端项目"
**When** 选择"全部项目"
**Then** 显示用户的所有任务（包括未绑���项目的任务）

### Requirement: 任务标题显示项目名

任务列表中的任务标题应显示为"项目名 - 任务名"格式（如果任务已绑定项目）。

#### Scenario: 已绑定项目的任务标题

**Given** 任务"修复登录 Bug"绑定到项目"前端项目"
**When** 在任务列表中显示
**Then** 标题显示为"前端项目 - 修复登录 Bug"

#### Scenario: 未绑定项目的任务标题

**Given** 任务"临时测试"未绑定任何项目
**When** 在任务列表中显示
**Then** 标题仅显示"临时测试"��无项目前缀）

#### Scenario: 长标题截断

**Given** 项目名"超级复杂的前端重构项目"，任务名"修复首页加载缓慢问题以及优化组件性能"
**When** 在窄屏设备上显示
**Then** 标题应用 CSS `text-overflow: ellipsis` 截断，鼠标悬停显示完整标题

### Requirement: 筛选器状态持久化（可选）

用户选择的项目筛选条件应在页面刷新后保留。

#### Scenario: 刷新后保留筛选

**Given** 用户选择了"前端项目"筛选器
**When** 刷新页面
**Then** 筛选器仍然选中"前端项目"，任务列表显示过滤后的结果
