## ADDED Requirements

### Requirement: 任务通知配置支持模板复用

系统 SHALL 支持将通知配置抽离为可复用模板，任务只选择模板而不重复录入完整参数。

#### Scenario: 任务选择已有通知模板

**Given** 系统中已存在名为“研发群通知”的通知模板
**When** 用户编辑任务通知设置
**Then** 可直接选择“研发群通知”模板
**And** 任务保存后引用该模板

#### Scenario: 模板修改影响后续通知

**Given** 多个任务引用同一个通知模板
**When** 管理员修改该模板的 webhook 或 secret
**Then** 后续这些任务发送通知时使用更新后的模板配置

### Requirement: 通知模板集中管理

系统 SHALL 提供通知模板的新增、编辑、删除与列表能力。

#### Scenario: 新增通知模板

**Given** 用户进入通知模板管理界面
**When** 填写通道类型、webhook、secret 和触发策略并保存
**Then** 系统创建新的通知模板

#### Scenario: 删除未被引用的模板

**Given** 某通知模板当前没有任何任务引用
**When** 用户删除该模板
**Then** 系统成功删除该模板

#### Scenario: 删除被任务引用的模板

**Given** 某通知模板仍被任务引用
**When** 用户尝试删除该模板
**Then** 系统拒绝删除或要求先解除任务引用
